# Security policy

## Supported version

Foljapp is pre-alpha. Security fixes target the current `main` branch and the
public deployment at [foljapp.pages.dev](https://foljapp.pages.dev/). Historical
commits and local generated artifacts are not maintained as supported releases.

## Report a vulnerability privately

Use GitHub's [private vulnerability reporting form](https://github.com/okturan/foljapp/security/advisories/new).
Private vulnerability reporting is enabled, so reports can be discussed and
fixed without publishing exploit details in an issue.

Include the affected route, script, or commit; the impact; reproducible steps;
and any suggested mitigation. Do not include working exploit details, tokens,
private corpus data, or personal information in a public issue.

Use a normal GitHub issue for linguistic inaccuracies, citation corrections,
browser-layout bugs, and feature requests that do not create a security impact.

## Security and data boundary

- The deployed website and API are read-only reference surfaces. They do not
  provide accounts, authentication, or public write operations.
- The deployment contains reviewed verb data and generated public examples. It
  does not contain raw downloaded corpora, local SQLite/Tantivy indexes, or the
  large reports generated by the corpus lab.
- Raw and gated corpora live under the ignored `.cache/` tree for local research.
  Some sources require accepting third-party terms and using your own access
  token. Never commit those tokens, restricted datasets, or derived local indexes.
- Corpus download and indexing commands perform intentional local filesystem and
  network work. Review source URLs, licenses, output paths, and available disk
  space before running them; generated evidence is not trusted automatically.
- GitHub Actions runs with read-only repository permissions and immutable action
  revisions. CodeQL covers Actions and JavaScript/TypeScript; Rust is gated by
  formatting, strict Clippy, and the corpus-indexer test suite.

Useful reports include cross-site scripting or content injection, unintended
file or data exposure, unsafe corpus/archive handling, path traversal, dependency
or workflow compromise, leaked credentials, and a way to make the read-only API
perform an unauthorized state change.

## Disclosure

Please allow time to reproduce and remediate a confirmed issue before public
disclosure. This project does not currently operate a bug-bounty program.
