## Why

Per-verb `sources` fields should accurately credit the linguistic authorities consulted for each entry. The corpus has accumulated a citation gap during tier-1/2/3 expansion: 100 verbs have a Husić paradigm cache file (`.cache/husic/<id>.jsonl`, parsed via `scripts/parse-husic-pdf.py`), and `verify-engine.ts` consults this cache during regression checks — but only 14 of the 100 cite `husic` in their `sources` field. The other 86 (41 in the active corpus + 45 cache-only-without-corpus) understate the linguistic basis when surfaced to users.

Specifically, 41 corpus verbs DO have Husić cache data and DO get verified against it during build, yet `/verb/<lemma>` shows only "Kaikki — Wiktionary" + "Manual entry". The user inspecting `fejoj` (Kaikki + Manual is correct, no Husić data exists) versus `bashkoj` (cache exists, but page shows the same Kaikki + Manual) cannot tell that Husić was actually consulted for the latter.

## What Changes

- **Add** `{ source: 'husic', reference: 'Husić 2002 — parsed cache (.cache/husic/<id>.jsonl)' }` to the `sources` array of every corpus verb where a `.cache/husic/<id>.jsonl` file exists AND the entry does not already cite Husić. Affects 41 verbs.
- **Add** a vitest in `apps/web/lib/corpus-husic-citations.test.ts` asserting: for every verb whose `id` matches a `.cache/husic/<id>.jsonl` file, its `sources` SHALL include a `husic` entry. This guards future drift — adding a cache file without a citation (or vice versa) breaks the build.
- **Bump** corpus to `0.1.2` (additive, non-breaking — no engine implications). Updates `scripts/build-corpus.ts:CORPUS_VERSION` and the regenerated `data/verbs/version.json`.
- **Update** `packages/engine/docs/sources.md` Husić-cache statistic from "99 of 204 verbs with Husić data" to reflect the citation-completeness improvement.

## Capabilities

The change extends `verb-corpus`. A new requirement codifies the citation-completeness invariant. No other capability touched.

## Impact

- **Code** — 1 new test file (`corpus-husic-citations.test.ts`); `scripts/build-corpus.ts` version constant bump.
- **Data** — 41 corpus verb JSON files modified (single source entry appended each); `data/verbs/version.json` regenerated; `data/verbs/_corpus.client.json` regenerated by `build:corpus`.
- **No engine, API, schema, or routing changes.**
- **Audience tier** — Researchers benefit most (accurate provenance). Learners see no change unless they read the Sources panel.

## Non-Goals

- **No retroactive uniparser citations.** uniparser is the engine's paradigm-class authority (engine-wide, documented in `sources.md`); per-verb attribution is fuzzier and would require lexicon cross-reference. Out of scope.
- **No citations for the 44 Husić cross-resolved verbs** that lack a direct cache file. Those rely on paradigm+glossary cross-resolution; a different citation mechanism (or no citation) is appropriate. Could be a follow-up.
- **No new `source` enum values.** The existing `'uniparser' | 'kaikki' | 'husic' | 'manual'` set is sufficient.
- **No reordering or deduplication of existing source entries** beyond appending the new husic citation.
- **No change to the displayed Sources panel UI** (`citations-footer.tsx` already renders all source types).
