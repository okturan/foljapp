# foljapp architecture

A one-page map of how the system fits together. Deep context and the
14-capability roadmap live in [`openspec/config.yaml`](../openspec/config.yaml);
this doc is the data-flow and operational picture.

## Two worlds

The repo is deliberately split into a **runtime surface** (what ships to
Cloudflare Pages) and a **corpus lab** (local evidence tooling that never
deploys). Keep the boundary sharp — the lab produces small checked-in
artifacts that the app consumes; the app never reaches into `.cache/`.

```
RUNTIME (deploys)                          CORPUS LAB (local only)
  apps/web/           Next.js UI + API       tools/corpus-indexer/  Rust scanner
  packages/engine/    conjugation engine     scripts/*corpus*       build/audit/report
  packages/data/      schemas + loaders      data/corpora/          download ledger only
  data/verbs/         verb corpus (JSON)     .cache/                raw corpora, caches,
  data/opus/          demo parallel index                          SQLite, reports (gitignored)
  apps/web/public/examples/  prebuilt example assets  ◄── generated by the lab
```

Rule of thumb: `apps/`, `packages/`, `data/verbs/`, `data/opus/examples.json`,
and `apps/web/public/examples/` must stay small enough to build and deploy.
Everything under `.cache/` (currently ~231G) is evidence, not assets.

## The engine (`packages/engine`)

Pure TypeScript, zero runtime deps, framework-independent. Albanian is
fusional — you can't derive forms from rules alone, so each verb carries
principal parts (present/aorist/participle stems) plus per-verb
`cellOverrides` and `flags`, and the engine applies a rules pipeline.

```
data/verbs/<lemma>.json ─► configure() ─► conjugate(id, opts) ─► { form, decomposition }
                                        ├► table(id)            (full paradigm)
                                        └► trace(id, opts)      (step-by-step derivation)
```

- **Voice flags** (`packages/engine/src/types.ts`): `noMiddlePassive`,
  `middlePassiveThirdPersonOnly`, `thirdPersonOnly` — explicit lexical
  knowledge, applied before paradigm dispatch in `conjugate.ts`.
- **Negation** (`compose/particle.ts`): optative/subjunctive/imperative negate
  with `mos`; subjunctive inserts it *after* `të` (`të mos punoj`); everything
  else uses `nuk`/`s'`.
- **Sources of truth** (priority order in `openspec/config.yaml`): uniparser
  grammar → Kaikki Wiktionary → Husić dictionary → UD treebanks.
- **Regression gate**: `npx tsx scripts/verify-engine.ts` compares every cell
  against Kaikki/Husić. Baseline is tracked in
  [`packages/engine/docs/sources.md`](../packages/engine/docs/sources.md) and
  must not regress. Standing mismatches are decomposed there (all are
  third-party source errors, not engine errors).

## The corpus lab pipeline

Answers "which generated verb forms are actually attested in real Albanian
text?" over ~1.45B candidate sentences from downloaded corpora. The Rust
indexer (`tools/corpus-indexer`, subcommands: `match`, `build-candidate-cache`,
`phrase-variant-stress`, `report-raw-coverage`, `trace-targets`,
`build-search-index`, `search-index`, `bench`) does the heavy scanning.

```
data/verbs ─►[build:corpus-targets]─► .cache/corpus-targets.json ─┐
                                                                  ▼
.cache/datasets/<corpus>  ─►[build:corpus-candidate-cache]─► split candidate cache
  (raw corpora, per resources.json)     (parquet/jsonl/xml/…)    .cache/corpus-candidate-shards/
                                                                  split-20260620/  (per-partition
                                                                  .norm/.rows/.tokens/.target-hits.zst)
                                                                  ▼
                          [scan:local-corpus:cached]─► .cache/corpus-local-full.sqlite  (retained examples)
                                                                  ▼
                          [audit:corpus-misses:full]─► .cache/corpus-missing-audit.json (hit/miss + UniParser)
                                                                  ▼
       [report:corpus-phrase-variants:all]─► variant-recovery report (parity-gated)
                                                                  ▼
                          [build:static-examples]─► apps/web/public/examples/<verbId>.json  ─► app
```

### Full-rescan runbook (order matters)

After any change that alters generated targets (engine/verb-data change) or
adds a corpus source, run `npm run rescan` (wrapper: `scripts/rescan.sh`;
`scripts/rescan.sh --plan` prints the steps). It runs, in this exact order:

1. `npm run build:corpus-targets` — regenerate targets from the engine.
2. `npm run build:corpus-candidate-cache` — build/refresh candidate-cache
   partitions **and target-hit sidecars**. New corpus partitions get parsed
   here; existing fresh partitions short-circuit.
3. `npm run scan:local-corpus:cached` — rebuild the retained-examples SQLite
   (requires the cache to exist — hence step 2 first).
4. `npm run audit:corpus-misses:full` — coverage + UniParser analysis + audit.
5. `npm run report:corpus-raw-coverage` and
   `npm run report:corpus-phrase-variants:all`.
6. `npm run build:static-examples` — regenerate the deployed example assets.

**Ordering traps (learned the hard way):**
- `audit:corpus-misses:full` regenerates `report:corpus-coverage` internally;
  don't run the audit against a stale coverage snapshot.
- Target-hit sidecars are refreshed by `build:corpus-candidate-cache`, **not**
  by the scan. If targets changed, run step 2 before the coverage/raw reports.
- The phrase-variant report is parity-gated: verify output invariance with
  `npm run report:corpus-phrase-variants:diff <baseline.json> <candidate.json>`.

### Adding a corpus source — THREE registration points

A source can be downloaded, ledgered, and reader-wired yet still silently
skipped. Adding one requires **all three**:
1. `data/corpora/resources.json` — entry with `status: "downloaded"`, a
   `localPath`, and a recognized `format`.
2. `tools/corpus-indexer/src/sources.rs` — the `source_kind` match arm
   (reuse `ParquetDir`/`Mc4JsonGzDir`/etc. — the reader usually already exists).
3. `tools/corpus-indexer/src/main.rs` — `ALL_SOURCE_IDS` **and**
   `alias_source`. `--sources=all` expands the curated ID list, not the
   ledger; a missing entry shows up as a suspiciously fast (~2s) no-op in
   `build-candidate-cache`. Watch stage timing.

## App runtime & deploy

```
data/verbs ─► packages/engine ─► apps/web (verb pages, /playground, /browse)
apps/web/public/examples/*.json ─► CorpusExamples panel (client fetch, no backend)
/api/examples ─► edge route: target derivation + OPUS parallel pairs only
```

- Conjugation tables are **client-rendered** (`ssr: false`) — a deliberate
  Pages-artifact-size tradeoff (see the `reference-pages` spec). The static
  shell + metadata are prerendered.
- Deploy: `npm run deploy:pages` (next-on-pages build → patch `_routes.json`
  to serve `/examples/*` statically → `wrangler pages deploy … --branch main`).
  The build pins `legacy-peer-deps` to survive wrangler/workers-types drift.

## Quality gates (must pass before archiving any change)

```
npm run typecheck   # strict TS across workspaces
npm run lint        # ESLint
npm test            # Vitest (engine + unit)
npm run build       # Next.js production build
npm run test:e2e    # Playwright (needs browsers)
CARGO_TARGET_DIR=.cache/cargo-target cargo test --manifest-path tools/corpus-indexer/Cargo.toml
npx tsx scripts/verify-engine.ts --check   # engine baseline gate (exits 1 on regression)
openspec validate <change> --strict
```

`verify-engine` can't run in CI (it needs the gitignored Kaikki/Husić
caches), so a pre-push hook enforces it locally. Enable once per clone:
`git config core.hooksPath scripts/hooks`. It runs `--check` and blocks a
push that drops below the baseline (skips cleanly if the caches are absent).
CI gates the rest: typecheck, lint, unit tests, build, and the Rust
corpus-indexer tests.

Every non-trivial change opens an OpenSpec change first
(`openspec/changes/`), lands green, then archives (`openspec/changes/archive/`).
Rust builds always use `CARGO_TARGET_DIR=.cache/cargo-target`.
